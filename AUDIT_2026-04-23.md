# Senior-Engineer Audit — financial-health-tracker

**Reviewer:** Claude (Opus), acting as a senior engineer.
**Date:** 2026-04-23.
**Scope:** `main` of https://github.com/marinav13/financial-health-tracker as cloned at `/Financial Health Tracker - For Github/`. HTML/CSS/JS, R + Python data pipelines, tests, GitHub Actions, Supabase integration.
**Ground rules honored:** public Supabase anon key not flagged, closure-data non-display not penalized, methodology placeholder not penalized, scraper fragility only flagged where handling is careless.

---

## TL;DR

The site works, the structure is professional, and the effort put into tests, runbooks, and refresh pipelines is above average for a journalist-run static project. But the code has the fingerprints of a lot of AI-assisted churn: at least one piece of documented AI truncation damage was "fixed" in a way that re-introduced broken CSS, a version pin was set to a lodash release that doesn't exist, and the same list/pagination/sort logic is copy-pasted across four JS files with small silent drifts. The data pipeline is the stronger half; the front-end is the weaker half.

**Overall: 71/100 — C+.** Good enough to hand to a professional reviewer with a short punch-list. Not good enough to hand to a beginner and walk away.

## Findings by category

### 1. Logic bugs and incorrect behavior

🔴 **`styles.css` lines 481–482 — broken CSS block (confirmed AI-damage residue).**
```
.section-copy strong {
  font-style: normal;
  font-weight: 700;
}
  color: #333;     ← orphan declaration
}                  ← extra closing brace
```
Browsers error-recover and skip the bad block, so the page still renders — but this is malformed source. Git blame shows commit `74c47432` truncated `styles.css`, commit `346c6d58` "restored" it, and a later commit added the `.section-copy strong { ... }` rule *above* the orphan without removing the orphan. Any real CSS linter (stylelint) would fail. Fix: delete the two orphan lines.

🟠 **`package.json` line 22 — `overrides: { "lodash": "4.18.1" }`.** That version does not exist on npm (lodash 4.x caps at `4.17.21`). `npm ci` against the committed lockfile currently works because the lockfile has `~4.17.23`, but the next `npm install` or `npm audit fix` will attempt to re-resolve overrides and fail with `ETARGET`. This reads like an AI-hallucinated version number. Likely intended: `"^4.17.21"` (or drop the override entirely if it was only added to silence a CVE).

🟠 **`js/app.js` — `syncTabs(options)` ignores `financialUnitid`.** `cuts.js`, `research.js`, and `school.js` each pass `{ financialUnitid: ... }` to `syncTabs`, but the function signature doesn't read that field. Either the callers are wrong or the implementation was half-refactored. Cross-tab linking silently falls back to the URL parameter.

🟡 **`js/accreditation.js` line ~63 — `OTHER_ACCREDITORS` constant is declared and never referenced.** Dead code. Either wire it into `renderLimitations` or delete it.

🟡 **`js/research.js` lines 472–473 — set `textContent` and immediately add `.is-hidden`.** Either hide before set, or don't set. Smells like a half-finished change.

🟡 **`js/research.js` lines 405–411 — `history.scrollRestoration = "manual"` followed by three `scrollTo(0,0)` calls.** A workaround that papers over an unidentified scroll-jump; worth finding the real cause rather than re-applying the fix.

🟡 **`js/cuts.js` title "(${school.cut_count})"** yields the literal string `(undefined)` when `cut_count` is missing. Guard with `?? 0`.

✅ **Non-issue I initially flagged and retracted.** `js/school.js:754, 942` reference `series.federal_grants_contracts_pell_adjusted_adjusted`. The double suffix looks like a typo but the field is real and produced deliberately by `scripts/build_ipeds_canonical_dataset.R:709` and `scripts/build_web_exports.R:710` (inflation-adjusted after Pell-adjustment). The naming is ugly but correct.

### 2. Edge cases and missing error handling

🔴 **`scripts/shared/accreditation_scrapers.R` (21 return sites).** Every parser returns `tibble::tibble()` on selector miss. A site redesign → zero-row scrape → pipeline continues → JSON republished with that accreditor's actions missing. No warning, no non-zero exit. Fix: when `nrow(result) == 0L` but the fetched HTML body is non-empty, log `warning()` with a message the refresh workflow can surface.

🟠 **`scripts/build_ipeds_canonical_dataset.R:90–92` — three `try(ensure_dictionary_archive(...), silent = TRUE)`.** If the dictionary download fails, the downstream `get_frequency_lookup()` silently returns empty factor maps and **every** decoded sector/ICLEVEL/HBCU/TRIBAL field becomes NA. Replace with `tryCatch` + explicit stop, or load a committed fallback dictionary.

🟠 **`scripts/import_supabase_institution_mapping.py:222` — `?limit=10000` with no pagination loop.** Supabase returns at most 10k in one response. If the institutions table ever exceeds that, rows are silently dropped. IPEDS has ~6k four-year institutions today, so this is future-tense — but it's a classic AI-written "looks right, breaks later" pattern.

🟠 **Closure import runs with `continue-on-error: true`** (`.github/workflows/refresh-ipeds-site-data.yml:129` and `refresh-ipeds-full.yml:127`). The Python script itself has good error messages; the workflow throws them away. Stale closure data gets republished and the workflow still shows green. Either remove the flag, or gate the subsequent web-export step on the closure step's `outcome`.

🟡 **`js/charts.js:safeChartColor` regex `/^#[0-9a-f]{3,8}$/i`** accepts invalid 5-digit or 7-digit hex. Tighten to `/^#([0-9a-f]{3}|[0-9a-f]{4}|[0-9a-f]{6}|[0-9a-f]{8})$/i`.

🟡 **`js/app.js` CSV download** calls `URL.revokeObjectURL` synchronously after `anchor.click()`. On some browsers this can cancel the download mid-flight. Wrap the revoke in `setTimeout(..., 0)` or `queueMicrotask`.

### 3. Data-flow and architecture

🟠 **Cross-file duplication with drift.** `cuts.js`, `research.js`, and `accreditation.js` each carry their own copies of: table filter+render, "other institutions" pagination block, and `setupPagination` / `setupOtherPagination`. The names are the same but the bodies have small differences (e.g. `research.js` triple-scrolls, `cuts.js` does not). This is the single biggest maintainability liability in the repo — fix something in one file and it drifts in the others. Consolidate into `app.js` as a `makeTableController(config)` factory.

🟠 **Three `normalize_name()` implementations** across `scripts/shared/accreditation_helpers.R` (Hawaiian okina + "the" stripping + "st" abbreviation), `scripts/shared/grant_witness_helpers.R` (shorter), and inline in `build_college_cuts_join.R`. Any match that relies on the R helper but not the Python analogue will silently miss its counterpart. Unify in `shared/utils.R`.

🟡 **Empty `<section class="school-meta-wrap"></section>`** in three HTML pages. Either put something there or remove the markup.

🟡 **Manual alias dict hardcoded** in `scripts/import_supabase_institution_mapping.py:73–100`. Every newly added school requires a code edit. Move to a CSV under `data_pipelines/`.

### 4. Performance

🟡 **`js/school.js` init** loads seven JSON files with `Promise.all`. Fine today, but each request is uncached and unversioned — a service-worker or a single bundled blob would be kinder on slow connections. Not urgent.

🟡 **`js/charts.js` tooltip rows** are built by wrapping values in `<span>`, the outer renderer strips spans, and then the fallback re-wraps. Two layers of escape/unescape for no net effect.

✅ Table pagination is already page-size-capped; no unbounded loops found.

### 5. Security

✅ `escapeHtml` is applied consistently where user-controlled text enters the DOM.
✅ Supabase anon key is public by design (noted, not penalized).

🟡 **`scripts/shared/accreditation_scrapers.R:61` — `iconv(x, from = "", to = "UTF-8", sub = "")`.** The empty `from` falls back to locale encoding; on a Windows runner this is Windows-1252 and will silently mojibake UTF-8 bytes. Force `from = "UTF-8"` and handle errors explicitly.

🟡 **`R download.file()` calls** (`scripts/shared/ipeds_row_builders.R:601`, `grant_witness_helpers.R:602`) have no explicit `timeout`/retry. A hanging IPEDS mirror can stall the refresh workflow for minutes. Switch to `httr2::req_perform` or set `options(timeout = 60)`.

🟢 **Anon-key placement.** It's fine that the key is public, but splitting the JWT across three lines in the Python script (lines 154–160) is obfuscation cosplay, not security. Consolidate and add a one-line comment stating it's a public anon key.

### 6. Dead / duplicated / drifted code

- `OTHER_ACCREDITORS` in `js/accreditation.js` (unused).
- `empty <section class="school-meta-wrap">` in three HTML files.
- Two nearly identical `escapeHtml` helpers inside `tests/test_accreditation_rendering.js` and `tests/test_research_security.js`.
- Repeated pagination/sort/filter code across `cuts.js`, `research.js`, `accreditation.js`.
- Inconsistent indentation (tabs vs two-space vs four-space) at specific lines in `cuts.js` (104, 225, 228, 248, 249, 258, 260) and `accreditation.js` (139, 205, 289). Classic partial-AI-edit fingerprint.
- `scripts/build_ipeds_dataset.R` (73 lines) is a thin shim that re-exports from `build_ipeds_canonical_dataset.R`. If it's only historical, delete it; if it's current, fold it in.

### 7. Tests

**What's good:**

- pa11y-ci at WCAG2AA against 8 pages + axe-core via Playwright exercising filtered/sorted/paginated states. That's well above average.
- Static-analysis smoke tests catch missing functions or broken string tables.
- R and Python helpers have `testthat` / `pytest` unit coverage for the small pure-function layer.

**What's missing (confidence gaps):**

- No end-to-end pipeline test. Nothing runs `collect → canonical → web_exports` and validates the JSON shape against a schema. A silent field rename anywhere in the R pipeline will only surface when a journalist notices something weird on the site.
- Accreditation filter input is not exercised (`test_accreditation_rendering.js` stubs `loadJson` and never types in `#accreditation-filter`).
- CSV exports are only checked for "produced"; contents are not asserted.
- `syncTabs` cross-page linking is checked for symbol existence but never driven with a URL parameter.
- Empty/partial data scenarios (school with missing years, school with no accreditation actions, school with zero cuts) are absent.
- No Firefox, Safari, or mobile viewport in Playwright config.
- `test_refresh_workflows.js` validates that YAML strings *contain* `timeout-minutes`; it does not validate that the values are sensible.
- Tests like `frontend-state.spec.js:35` assert only that a cell has non-empty text, not that it contains a valid state code.

**AI-test smells:**

- `test_accreditation_rendering.js` stubs `trackerApp.loadJson` — i.e. the test replaces the code under test with a fixture and then checks the fixture. Passing this test tells you almost nothing.
- `test_security_helpers.js` re-extracts helpers with `new Function(...)` and tests them in isolation, so integration bugs between those helpers and the real DOM won't show up.
- Duplicative `escapeHtml` declarations inside two test files.

### 8. Accessibility

- `index.html` has no `<h1>`. `masthead-title` is a `<div>`. axe-core with default rules will typically flag this under `page-has-heading-one`; pa11y-ci at WCAG2AA may not. Either way, a screen-reader user lands without a document heading.
- `cuts.html`, `research.html`, `accreditation.html` each ship an `<h1 class="is-hidden">Loading...</h1>` that is never un-hidden on the landing view, meaning the landing-page mode of each tab also has no visible or reachable h1.
- The custom `role="listbox"` + button-as-`role="option"` pattern for search results is nonstandard but workable; `aria-activedescendant` and Arrow-key handling are present.
- Skip link exists and targets `#main`; good.
- `aria-live="polite"` on table status regions; good.

Overall accessibility is better than average but two structural headings are missing and the test suite doesn't catch them because the pages technically *have* an h1 element — just a permanently hidden one.

## AI-generated damage and unfinished fixes

This is the pattern I want to name explicitly, because the user asked for it.

1. **Styles.css truncation + re-truncation.** Commit `74c47432` truncated the file; commit `346c6d58` "restored" it; a later commit added `.section-copy strong` rules; the orphan `color: #333; }` was left behind at lines 481–482. This is the clearest fingerprint of "AI cleaned up what it broke without actually reading the result."
2. **`lodash: 4.18.1` override.** A version that does not exist. Reads as a model completion that went one minor past reality.
3. **`syncTabs(financialUnitid)` drift.** Three call sites pass the option; the function doesn't consume it. Either a rename was only half-applied, or the feature was planned and never finished.
4. **Three divergent `normalize_name()`s.** Same semantic goal, three implementations. Two are probably copy-pasted-then-edited by an assistant.
5. **Scattered indentation inconsistencies** (spaces mixed with a stray tab, block shifted by 2 columns in the middle of a function). These are almost always partial-edit residues.
6. **Set-then-hide in `research.js` (472–473)** and triple-`scrollTo` (405–411). These are "I don't know why it isn't working, try again" edits rather than root-cause fixes.
7. **`continue-on-error: true` on the closure step.** In isolation this can be correct engineering; in context it masks a class of failure the rest of the workflow is trying to catch.
8. **Stubbed tests that pretend to test the real thing** (`test_accreditation_rendering.js` replacing `loadJson` with a fixture).

None of these are catastrophic individually. The pattern is the concern: it says the repo has been iterated on by a tool that completes locally without verifying globally.

## Why this grade is not inflated

I gave this project a **C+ (71)**, not a B or B-, on purpose:

- **Correctness has confirmed defects shipping to production.** Broken CSS, a non-existent npm version pinned, and a `syncTabs` parameter that's ignored are not stylistic preferences — they are bugs that would be flagged in any professional review.
- **The repo shows AI-editing damage that was never fully cleaned up.** A user asked for a restore; the restore was incomplete; nobody noticed. That is the exact failure mode a beginner maintainer will inherit.
- **Critical paths fail silently.** Scrapers returning empty frames, dictionary archives swallowed by `try(silent=TRUE)`, and `continue-on-error: true` on the closure import all share the same anti-pattern: green CI with wrong data. For a newsroom tool, that is worse than a crash.
- **Test breadth ≠ test depth.** Counting test files, this suite looks impressive. Reading them, a lot of them stub the code under test or assert on existence rather than behavior. The suite would not catch the bugs I found above.
- **The duplication across page controllers is real technical debt, not a stylistic quibble.** Three places must be kept in sync for every change; they are already out of sync.
- **Accessibility is above average but still has real issues** (missing/hidden h1 on landing pages) that the current a11y test configuration does not catch.

What kept it out of the D range:

- Data pipelines are broadly sound — column contracts, row-builders, and enrichment helpers are thoughtfully factored.
- Runbooks and workflows exist and are mostly correct; a beginner has somewhere to start.
- Security posture for a static site is appropriate.
- Testing *exists* at multiple layers (R, Python, JS smoke, Playwright e2e, pa11y-ci, axe-core). Shallow, but there.
- Nothing is catastrophically wrong. The site works. Users get real data.

A grade in the B range would require the logic bugs fixed, the dead/duplicate code consolidated, the silent-failure modes surfaced, and an end-to-end pipeline contract test. That is maybe two to three focused weeks of work for a single capable engineer, not a rewrite.
