IPEDS Financial Health Project
Full Walkthrough

This project rebuilds the college financial health interactive from source
files to the static site assets that live on the Hechinger website.

What this repo does

- collects and decodes IPEDS
- builds one raw-but-decoded institution-year dataset
- builds one authoritative canonical dataset
- joins the supporting domain tables from `data_pipelines/`
- writes the web exports and canonical workbook

Important closure note

This repo no longer scrapes federal closure data itself.

Instead:

- a separate closure workflow or repo prepares the closure data
- that workflow publishes cleaned closure outputs into a Google Sheet
- this repo imports those published outputs with `scripts/import_closure_sheet.py`

That is intentional. It keeps the interactive repo smaller, clearer, and easier
for a beginner to rerun.

Project folders

- `scripts/` contains the active build entrypoints
- `data_pipelines/` contains the non-IPEDS local rebuild inputs and intermediate joins
- `ipeds/raw/` contains the raw decoded institution-year CSV you rebuild locally when needed
- `ipeds/manifests/` contains the selected-file catalog and field-resolution audit you keep locally
- `ipeds/derived/` contains the canonical and extended IPEDS datasets you keep locally for rebuilds and reporting
- `ipeds/cache/` is the reusable local IPEDS download cache
- keep `ipeds/cache/` on your machine
- do not commit `ipeds/cache/` to Git
- `data/` contains the site JSON exports and indexes
- `workbooks/` contains the local article workbook export

Git policy for this repo:

- commit code, docs, workflows, and finished site assets in `data/`
- keep caches, raw files, intermediate pipeline outputs, and workbook files on your machine
- do not commit those rebuild-only files to Git

One-time R setup

Open RStudio and set the working directory to the project folder.

Run:

```r
user_lib <- Sys.getenv("R_LIBS_USER")
if (!nzchar(user_lib)) {
  user_lib <- file.path(path.expand("~"), "R", paste0("library-", getRversion()))
}
dir.create(user_lib, recursive = TRUE, showWarnings = FALSE)
.libPaths(c(user_lib, .libPaths()))

install.packages(c(
  "dplyr",
  "googlesheets4",
  "openxlsx",
  "purrr",
  "readr",
  "readxl",
  "stringr",
  "tidyr",
  "xml2"
))
```

How to rebuild everything from scratch

Step 1. Build the IPEDS dataset

```r
source("scripts/build_ipeds_dataset.R")
main(c("--start-year", "2014", "--end-year", "2024"))
```

This creates:

- `ipeds/raw/ipeds_financial_health_raw_2014_2024.csv`
- `ipeds/manifests/ipeds_financial_health_selected_file_catalog.csv`
- `ipeds/manifests/ipeds_financial_health_field_resolution_audit_2014_2024.csv`
- `ipeds/derived/ipeds_financial_health_canonical_2014_2024.csv`
- `ipeds/derived/ipeds_financial_health_dataset_2014_2024.csv`

Step 2. Build the supporting `data_pipelines/` joins

Run these in order:

```r
source("scripts/build_outcomes_join.R")
main()

source("scripts/build_college_cuts_join.R")
main()

source("scripts/build_accreditation_actions.R")
main()

source("scripts/build_grant_witness_join.R")
main()
```

Then run the Python profile inputs:

```bash
python scripts/build_hcm_level2.py
python scripts/import_closure_sheet.py --sheet "YOUR_CLOSURE_GOOGLE_SHEET_URL_OR_ID"
python scripts/build_federal_composite_scores.py
```

What the closure import does

It downloads the closure tabs from Google Sheets and refreshes:

- `data_pipelines/federal_closure/derived/running_closures.csv`
- `data_pipelines/federal_closure/derived/main_campus_closures.csv`
- `data_pipelines/federal_closure/derived/branch_campus_closures.csv`
- `data_pipelines/federal_closure/derived/mergers_consolidations.csv`
- `data_pipelines/federal_closure/derived/private_sector_federal_main_closures.csv`
- `data_pipelines/federal_closure/derived/closure_status_tracker_matches.csv`
- `data/closure_status_by_unitid.json`

If you want to test that import without using Google Sheets, point it at a
local folder of CSV exports:

```bash
python scripts/import_closure_sheet.py --from-dir path/to/closure_csv_exports
```

Step 3. Build the web exports

```r
source("scripts/build_web_exports.R")
main()
```

This creates:

- `data/downloads/full_dataset.csv`
- `data/schools_index.json`
- `data/schools/*.json`
- `data/accreditation.json`
- `data/research_funding.json`
- `data/college_cuts.json`
- `data/hcm2_by_unitid.json`
- `data/closure_status_by_unitid.json`
- `data/federal_composite_scores_by_unitid.json`
- `data/metadata.json`

Step 4. Build the workbook

```r
source("scripts/build_article_workbook.R")
main(c(
  "--input", "./ipeds/derived/ipeds_financial_health_dataset_2014_2024.csv",
  "--output", "./workbooks/ipeds_financial_health_article_workbook.xls"

```

- `/interactives/fitness/school.html?unitid=172264`

How to create and fill a Google Sheet for Looker Studio

If you already have a Google Sheet:

```r
source("scripts/publish_to_google_sheets.R")
main(c(
  "--input", "./data/downloads/full_dataset.csv",
  "--sheet", "YOUR_GOOGLE_SHEET_URL_OR_ID",
  "--tab", "looker_ready",
  "--email", "your.email@example.com"
))
```

If you want the script to create a new Google Sheet:

```r
source("scripts/publish_to_google_sheets.R")
main(c(
  "--input", "./data/downloads/full_dataset.csv",
  "--create", "IPEDS Financial Health Tracker",
  "--tab", "looker_ready",
  "--email", "your.email@example.com"
))
```

How to connect the Google Sheet to Looker Studio

1. Open Looker Studio.
2. Create a new data source.
3. Choose the Google Sheets connector.
4. Select the spreadsheet and the `looker_ready` tab.
5. Use the first row as headers.
6. Finish the connection.

How to automate the refresh

Use:

```bash
Rscript --vanilla scripts/annual_refresh_and_publish.R --start-year 2014 --end-year 2024
```

The scheduled site-data GitHub Action now imports closure outputs from the
published Google Sheet before rebuilding site data.

That means this repo does not need Google Cloud setup for closure publishing.

Spot-check list

- search works
- several school pages load
- cuts, accreditation, and research tabs populate
- closure and federal composite profile data populate
- the CSV download works
- the newest year appears where expected

If something is missing

If a later step fails because an input file is missing, go back one step and
rebuild the file it depends on.

Examples:

- if the web export fails, rebuild the outcomes join or IPEDS first
- if the workbook fails, rebuild the closure import or web export first
- if the closure import fails, check the Google Sheet URL and tab names first

Short version

1. Rebuild IPEDS.
2. Rebuild the supporting joins.
3. Import closure outputs from Google Sheets.
4. Rebuild the web exports.
5. Rebuild the workbook.
6. Copy the static site into `/interactives/fitness/`.
