# IPEDS Financial Health Project — Walkthrough

Rebuilds the college financial health interactive from federal data sources to the static files on the Hechinger website.

## What this repo does

1. Downloads and decodes IPEDS federal data for 2014–2024
2. Builds one cleaned canonical dataset
3. Joins in supporting data (accreditation actions, research cuts, college closures, etc.)
4. Writes the site JSON, CSV download, and school-level files

## Build everything from scratch

### Step 1. Build IPEDS

```r
source("scripts/build_ipeds_dataset.R")
main(c("--start-year", "2014", "--end-year", "2024"))
```

First time only — downloads ~2 GB of federal data and creates the cache. Later runs reuse the cache and only download what's new.

### Step 2. Build the supporting joins

```r
source("scripts/build_outcomes_join.R")
main()

source("scripts/build_college_cuts_join.R")
main()

source("scripts/build_accreditation_actions.R")
main()

source("scripts/build_grant_witness_join.R")
main()
```

Each join reads the canonical IPEDS dataset and adds one domain of external data.

### Step 3. Run Python profile imports

```bash
python scripts/build_hcm_level2.py
python scripts/import_closure_sheet.py --sheet "YOUR_GOOGLE_SHEET_URL_OR_ID"
python scripts/build_federal_composite_scores.py
```

The closure import downloads tabs from Google Sheets and writes them to `data_pipelines/federal_closure/derived/` and `data/closure_status_by_unitid.json`.

To test without Google Sheets:

```bash
python scripts/import_closure_sheet.py --from-dir path/to/closure_csv_exports
```

### Step 4. Build the site files

```r
source("scripts/build_web_exports.R")
main()
```

Reads all joined data, writes JSON indexes, school-level JSONs, and the CSV download.

### Step 5. Copy files to the website

Deploy these finished static files to `/interactives/fitness/`:

```
index.html, school.html, styles.css, js/, data/
```

The school page expects URLs like `/interactives/fitness/school.html?unitid=172264`.

## Rebuild just one section

If you only want to refresh one piece (e.g., research cuts) without touching anything else:

```bash
# 1. Rebuild research joins
Rscript --vanilla ./scripts/build_grant_witness_join.R --skip-usaspending-filter
Rscript --vanilla ./scripts/build_grant_witness_usaspending_sensitivity.R
Rscript --vanilla ./scripts/build_grant_witness_join.R

# 2. Rebuild site exports
Rscript --vanilla ./scripts/build_web_exports.R
```

This touches only research and the JSON output. It does not re-scrape accreditation, re-fetch cuts, or re-download IPEDS.

## Full automated refresh

```bash
Rscript --vanilla scripts/annual_refresh_and_publish.R --start-year 2014 --end-year 2024
```

This runs steps 2–4 in sequence. GitHub Actions runs this automatically every Monday at noon.

## If something breaks

The scripts tell you what they need. Work backward:

| Failure | Fix |
|---|---|
| Web export fails | Rebuild the join it depends on |
| Join fails | Rebuild the canonical IPEDS dataset |
| Closure import fails | Check the Google Sheet URL and tab names |
| IPEDS download fails | Check internet connection; the cache may be stale |

## Run the smoke tests

```bash
Rscript ./tests/run_shared_helper_smoke_tests.R
```
