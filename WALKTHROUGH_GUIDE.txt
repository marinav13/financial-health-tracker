IPEDS Financial Health Project
Full Walkthrough

This project rebuilds the college financial health interactive from raw source
files to the static site assets that live on the Hechinger website.

What this repo does

- scrapes and decodes IPEDS
- builds one raw-but-decoded institution-year dataset
- builds one authoritative canonical dataset
- joins the domain tables for cuts, accreditation, research, closures, HCM,
  and federal composite scores
- writes the web exports and workbook XML
- publishes the files that power the live interactive

Visual target

The finished interactive should match the structure and density shown in these
reference PDFs:

- `assets/interactive_examples/example_interactive.pdf`
- `assets/interactive_examples/example_cuts.pdf`
- `assets/interactive_examples/example_accreditation.pdf`
- `assets/interactive_examples/example_research.pdf`

Use them as the design target for the website handoff:

- the landing page should feel like a guided search tool, not a generic blog
- school pages should start with a clear profile summary and then move into
  chart sections and evidence sections
- the Cuts, Accreditation, and Research tabs should keep the same card-based
  structure and not collapse into plain tables
- the page should feel clean and readable, but still information-dense

Project folders

- `scripts/` contains the build entrypoints
- `scripts/ipeds/` contains the real IPEDS pipeline
- `scripts/shared/` contains the shared helper functions
- `ipeds/raw/` contains the raw decoded institution-year CSV
- `ipeds/cache/` contains downloaded IPEDS zips and extracted caches
- `ipeds/manifests/` contains the file catalog, column contract, and audit files
- `ipeds/derived/` contains the canonical and extended IPEDS datasets
- `data_pipelines/` contains the domain joins and federal closure outputs
- `data/` contains the site JSON exports and indexes
- `data/downloads/` contains the site-wide CSV download
- `data/schools/` contains the school-level JSON files
- `workbooks/` contains the workbook XML
- `docs/` contains the handoff notes

One-time R setup

Open RStudio and set the working directory to the project folder.

Run:

```r
user_lib <- Sys.getenv("R_LIBS_USER")
if (!nzchar(user_lib)) {
  user_lib <- file.path(path.expand("~"), "R", paste0("library-", getRversion()))

dir.create(user_lib, recursive = TRUE, showWarnings = FALSE)
```

How to rebuild everything from scratch

Step 1. Build the raw IPEDS dataset

```r
source("scripts/build_ipeds_tracker_dataset.R")
main(c("--start-year", "2014", "--end-year", "2024"))
```

This downloads the IPEDS source files into `ipeds/cache/downloads/` and
creates:

- `ipeds/raw/ipeds_financial_health_raw_2014_2024.csv`
- `ipeds/manifests/ipeds_financial_health_selected_file_catalog.csv`
- `ipeds/manifests/ipeds_financial_health_field_resolution_audit_2014_2024.csv`
- `ipeds/derived/ipeds_financial_health_canonical_2014_2024.csv`
- `ipeds/derived/ipeds_financial_health_dataset_2014_2024.csv`

Step 2. Build the domain joins and profile tables

Run these in order:

```r
source("scripts/build_outcomes_join.R")
main()

source("scripts/build_college_cuts_join.R")
main()

source("scripts/build_accreditation_actions.R")
main()

source("scripts/build_grant_witness_join.R")
main()
```

```powershell
python scripts/build_hcm_level2.py
python scripts/build_closure_status.py
python scripts/build_closure_outputs.py
python scripts/build_federal_closure_opeid_panel.py
python scripts/build_federal_closure_style_ipeds_closures.py
python scripts/build_federal_composite_scores.py
```

Step 3. Build the web exports

```r
source("scripts/build_web_exports.R")
main()
```

This creates:

- `data/downloads/full_dataset.csv`
- `data/schools_index.json`
- `data/schools/*.json`
- `data/accreditation.json`
- `data/research_funding.json`
- `data/college_cuts.json`
- `data/hcm2_by_unitid.json`
- `data/closure_status_by_unitid.json`
- `data/federal_composite_scores_by_unitid.json`
- `data/metadata.json`

Step 4. Build the workbook

```r
  "--output", "./workbooks/ipeds_financial_health_article_workbook_r.xml"
```

How the Google Sheets publishing works

The recommended setup is:

- build data in R
- publish only the site-ready CSV to Google Sheets
- connect Looker Studio to that sheet

Do not use the raw CSV as the main Google Sheet source unless you really need
it.

Recommended source:

- `data/downloads/full_dataset.csv`

How to create and fill a Google Sheet

If you already have a Google Sheet:

```r
source("scripts/publish_to_google_sheets.R")
main(c(
  "--input", "./data/downloads/full_dataset.csv",
  "--sheet", "YOUR_GOOGLE_SHEET_URL_OR_ID",
  "--tab", "looker_ready",
  "--email", "your.email@example.com"
))
```

If you want the script to create a new Google Sheet:

```r
source("scripts/publish_to_google_sheets.R")
main(c(
  "--input", "./data/downloads/full_dataset.csv",
  "--create", "IPEDS Financial Health Tracker",
  "--tab", "looker_ready",
  "--email", "your.email@example.com"
))
```

Weekly closure status spreadsheet

The closure status sheet is a separate Google Sheet for the current federal
school-file closure matches.

Use it when you want the freshest closure status without reopening the larger
historical closure workbook.

Publish it with:

```r
source("scripts/publish_closure_status_sheet.R")
main(c(
  "--sheet", "YOUR_CLOSURE_GOOGLE_SHEET_URL_OR_ID",
  "--tab", "tracker_closure_status",
  "--summary-tab", "tracker_closure_summary"
))
```

This sheet uses:

- `data_pipelines/federal_closure/derived/closure_status_tracker_matches.csv`
- `data/closure_status_by_unitid.json`

The weekly closure workflow downloads the current federal closed-school report
for you. You do not need to place the ZIP in the repo by hand.
If you automate the sheet in GitHub Actions, use Google Cloud Workload Identity
Federation instead of a JSON key file, then share the spreadsheet with the
service account email. If you already have a local Google Cloud credentials
file, the script will also read `GOOGLE_APPLICATION_CREDENTIALS` automatically.

How to connect the Google Sheet to Looker Studio

1. Open Looker Studio.
2. Create a new data source.
3. Choose the Google Sheets connector.
4. Select the spreadsheet and the `looker_ready` tab.
5. Use the first row as headers.
6. Finish the connection.

Important sharing and credentials setting

If you plan to share the report with other people, set the data source to
`Owner's credentials`, not `Viewer's credentials`.

Why this matters:

- `Owner's credentials` lets viewers see the report using the data source
  connection owned by the report creator
- `Viewer's credentials` can cause outside users to hit vague system errors

How to automate the refresh

Use any scheduler you prefer. The command should look like this:

```powershell
Rscript --vanilla scripts/annual_refresh_and_publish.R --start-year 2014 --end-year 2024 --sheet YOUR_GOOGLE_SHEET_URL_OR_ID --tab looker_ready --email your.email@example.com
```

Use the same Google Sheet every time.

If something is missing

If a later step fails because an input file is missing, go back one step and
rebuild the file it depends on.

Examples:

- if the web export fails, rebuild the outcomes join or the IPEDS tracker first
- if the workbook fails, rebuild the web export first
- if a closure or composite-score script fails, make sure the raw source files
  are in `data_pipelines/federal_closure/raw/` or `ipeds/cache/downloads/`

Short version

1. Rebuild IPEDS.
2. Rebuild the domain joins.
3. Rebuild the web exports.
4. Rebuild the workbook.
5. Copy the static site into `/interactives/fitness/`.
