IPEDS Release-Cycle Update Checklist

This project should be refreshed three times a year, tied to the IPEDS release cycle.

2024-25 release schedule

- Fall release: September 23, 2025
- Winter release: December 9, 2025
- Spring release: January 6, 2026

What those releases mostly affect

- Fall surveys: Institutional Characteristics, Completions, 12-month Enrollment
- Winter surveys: Admissions, Graduation Rates, Student Financial Aid, Outcome Measures, Cost
- Spring surveys: Fall Enrollment, Finance, Human Resources, Academic Libraries

Recommended run dates

- September 24
- December 10
- January 7

One-command refresh

```r
source("scripts/annual_refresh_and_publish.R")
main(c(
  "--start-year", "2014",
  "--end-year", "2024",
  "--sheet", "YOUR_GOOGLE_SHEET_URL_OR_ID",
  "--tab", "looker_ready",
  "--email", "your.email@example.com"
))
```

Manual step-by-step refresh

1. Rebuild the raw file

```r
source("scripts/build_ipeds_tracker_dataset.R")
main(c("--start-year", "2014", "--end-year", "2024"))
```

2. Rebuild the Looker and reporting files

```r
source("scripts/build_looker_ready_metrics.R")
main()
```

3. Rebuild the workbook

```r
source("scripts/build_article_workbook.R")
main(c(
  "--input", "./reporting/ipeds_financial_health_reporting_2014_2024.csv",
  "--output", "./workbooks/ipeds_financial_health_article_workbook_r.xml"
))
```

4. Publish to Google Sheets

```r
source("scripts/publish_to_google_sheets.R")
main(c(
  "--input", "./looker_studio/ipeds_financial_health_looker_ready_2014_2024.csv",
  "--sheet", "YOUR_GOOGLE_SHEET_URL_OR_ID",
  "--tab", "looker_ready",
  "--email", "your.email@example.com"
))
```

If you want to test the weekly closure status sheet locally, you can point the
publisher at the downloaded weekly files like this:

```r
source("scripts/publish_closure_status_sheet.R")
main(c(
  "--weekly-zip", "./data_pipelines/federal_closure/raw/weekly_closed_school_search/latest.zip",
  "--weekly-metadata", "./data_pipelines/federal_closure/raw/weekly_closed_school_search/latest.json",
  "--sheet", "YOUR_CLOSURE_GOOGLE_SHEET_URL_OR_ID"
))
```

When you run this manually, the script will use `GOOGLE_APPLICATION_CREDENTIALS`
if it is already set. The GitHub Action does this automatically, so you do not
need to fill those paths in by hand for the scheduled weekly run.

5. Open Looker Studio and confirm the new data is present

- the Google Sheet tab refreshed
- the newest year appears in scorecards and charts
- sentence cards filtered to `year = 2024` still work

Spot-check before publishing

- one public flagship
- one private nonprofit
- one for-profit
- one school with high international enrollment
- one school with large state support
- one school with high federal grants/contracts dependence

Check the newest year for:

- enrollment
- staffing
- revenue
- expenses
- loss
- net tuition per FTE
- transfer-out rate
